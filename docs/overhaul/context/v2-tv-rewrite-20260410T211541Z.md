Task statement

Plan CrispyTivi v2 from scratch on a clean branch using Flutter MV only and Rust FFI for controller/business logic. Produce consensus planning artifacts only: PRD, test spec, execution plan, and AGENTS amendments. No implementation yet.

Desired outcome

- Full app-wide v2 plan grounded in repo structure and current workspace constraints.
- Fully navigable shell/stub app planned for all screens and flows.
- Vertical delivery order locked by domain.
- Canonical source capability model, media envelope, windowed primitive system, and navigation/input model defined.
- AGENTS guidance updated later to enforce performance-first, TDD-first, DDD/SOLID/LOB/DRY ordering, MV-only Flutter, and player-last investigation gate.

Known facts/evidence

- `app/flutter/lib/.gitkeep` indicates Flutter app code is intentionally empty on this branch.
- `rust/Cargo.toml` defines a workspace with no current members and shared Rust dependencies plus shared IPTV crates (`crispy-m3u`, `crispy-xtream`, `crispy-stalker`, `crispy-xmltv`, `crispy-catchup`, `crispy-media-probe`, `crispy-iptv-types`, `crispy-iptv-tools`, `crispy-stream-checker`).
- Shared Rust crates already exist under `rust/shared/`.
- Design-system docs and existing workflow surfaces exist in `design/docs/` and `docs/overhaul/plans/`.
- Existing repo scripts already encode DDD/SOLID/DRY audits for Dart and Rust:
  - `scripts/dart/audit_ddd_solid_dry.sh`
  - `scripts/rust/audit_ddd_solid_dry.sh`
- User locked a TV-first, scale-not-reflow, fake-scroll/windowed shell with remote/keyboard/gamepad-first navigation design.

Constraints

- No implementation in this phase.
- Flutter code: MV only, no controller/business logic.
- Rust FFI owns controllers/business/domain orchestration and provider translation.
- Priorities: performance/RAM/CPU, TDD, DDD, SOLID, LOB, DRY.
- App must be fully navigable by remote/keyboard/gamepad from the start.
- All screens use same fake-scroll/windowed primitives from day 1.
- Home shell + all stubs planned first; real delivery proceeds one vertical at a time.
- Player is last major implementation after dedicated investigation/subplan.
- Flutter tokens authoritative; Penpot mirrors; Widgetbook required from start.
